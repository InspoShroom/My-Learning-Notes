# 使用类与对象

​	第二节涵盖了 C++ 中常用的内置类型（如 `std::string` 与文件 I/O）、内存模型与变量分类、动态内存分配、引用（References）以及 `const` 常量的深度解析。

## 标准库类型与文件输入输出

### 字符串类 `std::string`

C++ 的 `std::string` 相比 C 语言的 `char[]` 提供了更安全、更友好的接口。

* 定义与初始化：
  
    ??? example "点击查看代码"
        ```cpp
        #include <string>
        using namespace std;
        string str1;              // 默认初始化为空字符串 ""
        string str5 = ("good")	  // 显式调用构造函数
        string str2 = "hello";    // 字面量初始化
        string str3(str2);        // 拷贝构造
        string str4(5, 'a');      // 构造为 "aaaaa"
        
        ```
  
  ​	关于第二、三这两种初始化方式，如果只想初始化一个字符，写成`('s')` 是错误的，使用双引号 `("s")` 才可以。实际上这也是双引号和单引号的区别，单引号括起来的是字符字面量，而双引号括住的是字符串字面量。
  
* 常用操作：
     * **赋值与拼接**：支持直接通过 `=` 赋值（C 语言中对字符数组赋值非法），支持通过 `+` 或 `+=` 进行拼接。
     * **子串提取**：`substr(pos, len)` 返回从 `pos` 开始、长度为 `len` 的子串。
     * **修改字符串**：提供 `assign()`、`insert()`、`erase()`、`append()`、`replace()` 等成员函数。
    * **查找**：`find(target)` 返回目标字符串在原串中首次出现的位置，若未找到则返回 `string::npos`。

### 文件 I/O (`<fstream>`)

C++ 使用文件流对象来读写文件：

* `ifstream`（输入文件流）：用于从文件中读取数据。
* `ofstream`（输出文件流）：用于向文件中写入数据。

??? example "点击查看代码"
    ```cpp
    #include <fstream>
    #include <string>
    
    // 写入文件
    std::ofstream File1("C:\\test.txt");
    File1 << "Hello world" << std::endl; 
    File1.close();
    
    // 读取文件
    std::ifstream File2("C:\\test.txt");
    std::string str;
    File2 >> str; // 默认以空格或换行符分割读取
    File2.close();
    ```
我们可以对比理解读写过程: 

1. `ofstream` 和 `ifstream` 实际上都是在创造一个流对象, 并与目标文件进行绑定.
2. 使用操作符流输出`<<`以及流插入`>>` 进行读写时, 类似于将文本进行了"重定向", 用平常的`cout << "Hello World"` 做对比. 这里的`cout`就是对应到终端/控制台/console的流对象。同样的，`cerr`对应到错误输出流对象。

## C++ 内存布局与变量分类

C++ 程序在运行时，内存会被划分为不同的物理区域，以便高效管理数据存取：

| 内存区域 | 存储的数据类型 | 特点 |
| :--- | :--- | :--- |
| **栈（Stack）** | 局部变量、函数形参、函数调用上下文 | 由编译器自动分配和释放。空间较小，速度极快。遵循后进先出（LIFO）原则。 |
| **堆（Heap）** | 动态分配的变量（如通过 `new` 或 `malloc` 申请的内存） | 程序员手动申请和释放（如果不释放会导致内存泄漏）。空间极大，访问速度稍慢。 |
| **全局数据区/静态区（Global Data/Static Area）** | 全局变量、静态全局变量、静态局部变量 | 在程序编译时就已分配好，在程序运行的整个生命周期内都有效。 |

1. 如果全局变量要在其他文件中使用，那么其`.h`文件中声明时，需要用到`extern`关键字。
2. `extern`关键字仅用来声明，不负责定义，定义是在`.cpp`文件中完成的。按照定义的判断条件，必须发生内存分配才叫定义
3. 其他文件使用时只需要包含那个`.h`文件即可。

### static 关键字的特殊规则

`static` 修饰符在全局和局部变量中具有截然不同的作用语义：

* **局部事物（静态局部变量）**：实现**持久性（Persistence）**。静态局部变量在多次调用该函数之间保持其值，只在第一次被访问时进行初始化，生存期延长至整个程序结束。
* **全局事物（静态全局变量/静态函数）**：实现**访问限制（Access Restriction）**。静态全局变量/函数只在当前 `.cpp` 编译单元可见，禁止来自外部编译单元的访问，防止全局命名污染。

??? example "点击查看代码"
    ```cpp
    // 示例：extern 与 static 链接性
    // file1.cpp
    int global_val = 100;        // 全局变量定义
    static int static_val = 200; // 静态全局变量（仅在 file1.cpp 内可见）
    
    // file2.cpp
    extern int global_val;       // 声明外部定义的 global_val
    // extern int static_val;    // 错误！static_val 无法跨文件访问
    ```

## 指针与动态内存分配

### 指针操作对象
* `&` 运算符获取对象的内存地址。
* `*` 运算符解引用以获取指针指向的对象本身。
* `->` 运算符用于通过指针访问对象的成员函数或变量（例如 `ps->length()` 等价于 `(*ps).length()`）。

!!! warning "空指针与野指针"
    `string s;` 声明并默认构造了一个合法的空字符串。
    `string *ps;` 仅定义了一个指针变量，其值是随机的内存地址（野指针）。在未将有效地址赋值给它（或通过 `new` 申请内存）之前解引用野指针，会导致程序崩溃。

### 动态内存分配 `new` 与 `delete`
`new` 和 `delete` 是 C++ 中的运算符，相比 C 语言的 `malloc` 和 `free`，它们会自动调用类对象的构造函数与析构函数。

??? example "点击查看代码"
    ```cpp
    // 单个对象分配与释放
    int* p = new int;          // 堆上分配 4 字节 int
    delete p;                  // 释放
    
    Student* q = new Student();// 申请空间并自动调用构造函数
    delete q;                  // 自动调用析构函数并释放
    
    // 动态数组分配与释放
    int* a = new int[10];      // 堆上分配 10 个 int 空间
    delete[] a;                // 释放整个数组
    
    Student* r = new Student[10];
    delete[] r;                // 自动对数组中的每个对象调用析构函数，再释放内存
    ```

!!! info "new/delete 的配对黄金法则"
    * 必须保证通过 `new` 申请的内存最终通过 `delete` 释放，防止内存泄漏。
    * 如果申请时使用了 `[]`（如 `new Type[]`），释放时**必须**使用 `delete[]`。如果混用（如使用 `delete r;` 释放数组），编译器将只析构数组的第一个元素，从而导致后续元素析构函数未执行，甚至引发堆内存损坏。
    * 对空指针（`nullptr`）执行 `delete` 操作是安全的，不会发生任何事情。

## 引用（References）

引用是 C++ 引入的全新数据类型，它本质上是为一个已存在的变量起**别名（Alias）**。

### 声明与规则
??? example "点击查看代码"
    ```cpp
    int X = 47;
    int& Y = X; // Y 是 X 的引用，它们共同指向同一块内存空间
    Y = 18;     // 修改 Y，X 也会被修改为 18
    ```

1. **必须初始化**：引用在声明时必须绑定到一个已存在的变量，不能写 `int& y;`。
2. **绑定不可更改**：引用一旦建立绑定，在运行时就永远指向该对象，无法改指其他对象（这与指针可以随意更改指向不同）。
3. **目标必须有明确位置（左值）**：引用不能直接绑定到临时计算结果（右值）上。例如 `void func(int &); func(i * 3);` 是非法的。
> * 左值：拥有明确的内存地址，可以使用 `&` 来取地址。
> * 右值：一个临时的值，生命周期短，表达式结束就被摧毁，不能用 `&` 取地址。



### 指针 vs 引用

| 特性 | 指针（Pointer） | 引用（Reference） |
| :--- | :--- | :--- |
| **是否可为空** | 可以为 `nullptr` | 必须绑定有效对象，不能为空 |
| **独立性** | 是一个独立的变量（保存地址值） | 只是被引用变量的别名，不占用独立物理存储 |
| **可重新赋值** | 可以随时改变指向的地址 | 绑定关系终身不变 |

### 引用限制
* **不存在“引用的引用”**（例如 `int&&` 在 C++11 中被定义为右值引用，但不能建立引用的引用逻辑别名）。
* **不存在“指向引用的指针”**（例如 `int&* p;` 非法），但“指针的引用”是合法的（例如 `int*& p;`）。
* **不能定义“引用数组”**（例如 `int& arr[10];` 是非法的，因为数组元素必须是独立且连续排布的实体）。

## const 常量

### 编译时常量与运行时常量
* **编译时常量**：例如 `const int bufsize = 1024;`。编译器在编译期将符号表中的值直接替换，不为其分配内存。其值必须在声明时初始化。
* **运行时常量**：例如通过输入获取的 `const int size = x;`。虽然 `size` 也是 `const`（无法修改），但其值在编译期未知，因此在标准 C++ 中无法用来作为普通静态数组的长度（如 `double arr[size]` 非法）。

!!! note "聚合类型的 const"
    对于数组或结构体等聚合类型，使用 `const` 修饰时（例如 `const int i[] = {1, 2, 3};`），系统**会**为其分配内存空间。因为其内容保存在内存中，编译器在编译期无法直接查询其值，因此其元素同样不能作为编译时常量使用（如 `float f[i[2]];` 非法）。

### 指针与 const 的结合
通过观察 `const` 关键字在 `*` 的哪一侧，可以清晰区分常量指针与指针常量：

??? example "点击查看代码"
    ```cpp
    const char* p = "ABCD"; // 1. 常量指针（指向常量的指针）
    // const 在 * 左侧，修饰指向的字符数据。
    p++;                    // OK：指针本身的指向可以改
    *p = 'b';               // ERROR：指向的内容不能改
    
    char* const q = "abc";  // 2. 指针常量（指针本身是常量）
    // const 在 * 右侧，修饰指针变量 q。
    *q = 'c';               // OK：允许修改指向的字符数据
    q++;                    // ERROR：不能修改指针 q 本身的值（指向的地址）
    
    // 同理：
    const string* ps;       // 指向 const string 的指针，等价于 string const* ps
    string* const ps;       // 指向 string 的 const 指针
    ```

### 指针的 const 转换（权限缩小规则）
在传递指针或引用时，C++ 遵循**权限可缩小、不可放大**的原则：

* 可以将非 `const` 变量的地址赋给指向 `const` 的指针（隐式安全转换）。
* **绝对不允许**将 `const` 变量的地址赋给普通非 `const` 指针。例如下例：

    ??? example "点击查看代码"
        ```cpp
        const int ci = 3;
        int* ip = &ci; // ERROR！否则可以通过 *ip = 5 修改常量 ci 的值，破坏 const 约束。
        ```

### 按值传递/返回 vs 按地址传递/返回 const
* **按值传递/返回 const**：
  * 形参 `void f(const int i)`：限制在函数体内部无法修改参数副本，但对于外部实参没有实质安全性改善。
  * 返回值 `const int f()`：对于内置类型而言几乎无意义，因为返回值是以临时右值形式拷贝出去的。
* **按地址（指针或引用）传递/返回 const**：
  * 性能优化的核心方案。如果函数需要访问一个大型对象，直接按值传递会带来巨大的构造/拷贝开销；传递指针或引用只需要 8 字节（64位系统）。
  * 为了防止函数内部意外篡改传入的对象，应当尽可能使用 `const` 限制：
  
    ??? example "点击查看代码"
        ```cpp
        void printLargeObject(const MyLargeClass& obj); // 既高效又安全
        ```
