    第十一章 线程级并行与缓存一致性

    本章分析了多处理器存储架构分类，讨论了并行度与通信开销对性能的影响，系统梳理了监听一致性协议（MSI/MESI/MOESI/MESIF）、真伪共享不命中判别以及分布式目录协议机制。

    多处理器架构与并行阻碍

    1. 存储架构分类

    • 对称多处理器（Symmetric Multiprocessor, SMP）：共享单个中心化主存。所有核到主存具有相同的访问延迟，又称均匀访存（Uniform Memory Access, UMA）架构，扩展规模通常在 32 核以内。
    • 分布式共享内存（Distributed Shared Memory, DSM）：物理主存分布在各个计算节点上。不同核心访问不同物理位置的主存耗时不同，又称非均匀访存（Non-Uniform Memory Access, NUMA）架构。扩展性好，但节点间通信极其复杂。

    2. 并行编程的阿姆达尔定律算例

    • 算例 1：最大串行比例 使用 100 个处理器，要实现 80 倍的加速比，求程序中最大允许的串行比例（\(\text{Fraction}_{\text{seq}}\)）。 $\(\text{Speedup} = \frac{1}{\text{Fraction}_{\text{seq}} + \frac{1 - \text{Fraction}_{\text{seq}}}{100}} = 80 \implies \text{Fraction}_{\text{seq}} = \frac{1 - 80 \times 0.01}{80 \times 0.99} \approx 0.25\%\)$ 即串行部分不能超过 0.25%，说明多核并行对程序串行度极其敏感。
    • 算例 2：多级核数使用占比 应用程序可使用 1、50 或 100 个处理器。已知 95% 的时间使用全部 100 个处理器。要在 100 处理器上获得 80 倍的总体加速比，在剩余 5% 的时间内，有多大比例（\(f\)）必须使用 50 个处理器（其余时间用 1 个）？ $\(\text{Speedup} = \frac{1}{\frac{0.95}{100} + \frac{0.05 \times f}{50} + \frac{0.05 \times (1 - f)}{1}} = 80 \implies 0.0595 - 0.049f = \frac{1}{80} \implies f \approx 95.9\%\)$ 即在剩余的 5% 时间内，必须有 95.9% 的比例使用 50 处理器，其余 4.1% 使用单处理器。

    3. 远程通信延迟定量算例

    • 算例 3：32 核多处理器，访问远程内存延迟为 100ns。处理器主频 4.0 GHz（时钟周期为 0.25ns），基础 CPI 为 0.5。若发生 0.2% 的远程访存引用，求系统变慢的倍数。
    • 远程访存的周期数（Remote Cost） = \(100\text{ns} / 0.25\text{ns} = 400\) 周期。
    • 实际运行 CPI = \(\text{Base CPI} + \text{Remote Miss Rate} \times \text{Remote Cost} = 0.5 + 0.2\% \times 400 = 1.3\)。
    • 变慢倍数 = \(1.3 / 0.5 = 2.6\) 倍（无通信比有通信快 2.6 倍），表明远程访存延迟对多核性能有致命拖累。

    缓存一致性概念与分类

    • 缓存一致性（Cache Coherence）：定义了对同一个物理地址的读写顺序，保证读操作能够返回最近写入的数据。
    • 内存相容性（Memory Consistency）：定义了对多个不同物理地址的读写操作在全局范围内的可见顺序与时间。

    1. 一致性（Coherence）三大属性

    1. 程序顺序保留（Program Order）：核心 P 写入 X 随后读取 X，期间无其他核写，则读操作必须返回 P 刚写入的值。
    2. 写传播（Write Propagation）：核心 P 写入 X 之后，其他核心在足够长的时间后进行的读操作必须能返回该新值。
    3. 写串行化（Write Serialization）：所有核心对同一个物理地址 X 的两次写入，必须被所有核心以相同的顺序观察到。

    2. 一致性控制策略

    • 写失效（Write Invalidate）：核心写入数据时，广播失效信号使其他 Cache 持有的副本失效。写入前必须获得排他性（Exclusive Access）。
    • 写更新/广播（Write Update/Broadcast）：核心写入数据时，广播新值并强制更新其他所有 Cache 持有的副本。优点是下次读命中快，但极度消耗总线带宽。

    MSI 协议及其高级扩展

    监听协议（Snooping）依赖共享总线广播状态。其核心是 MSI 状态机：

    1. MSI 协议三状态

    • M（Modified，已修改）：块数据已被本地修改，是最新的，且与主存不一致（主存已过期）。块在当前 Cache 独占。
    • S（Shared，共享）：块可能在多个 Cache 共享，且与主存数据一致。数据只读。
    • I（Invalid，失效）：当前块内无有效数据。

    2. MSI 状态转换矩阵

    • 本地处理器请求：PrRd（读）、PrWr（写）。
    • 总线活动：BusRd（读数据）、BusRdX（独占读，申请写块并使他人失效）、BusUpgr（升级，S 变 M 并使他人失效）、Flush（将脏块数据写回主存或传送给请求者）。

    3. MSI 高级扩展协议

    • MESI 协议：引入 E（Exclusive，独占且干净）状态，表示块只存在于当前单个 Cache 且与主存一致。
    • 优势：当核心需要写入 E 状态的块时，可以直接转为 M 状态，无需在总线上广播失效信号，节约大量写带宽。
    • MOESI 协议：引入 O（Owned，所有者）状态，表示该块被多个核心共享，但主存数据已过期。
    • 优势：当 M 状态的块遭遇其他核读 miss 时，直接转为 O 状态并向请求核发送数据，无需将脏块数据立即写回主存，只有当 O 状态的块被置换（Eviction）时才写回主存。
    • MESIF 协议：引入 F（Forward，转发）状态。在多个 Shared 状态的共享核中，指定唯一的 F 核去响应其他核心的 Read 请求，避免多个 Shared 核心同时在总线上抢占响应。最晚读取该块的核心被设为 F 状态。

    真共享不命中与伪共享不命中

    • 真共享不命中（True Sharing Miss）：由于数据在核心间真正被传递和修改造成的失效不命中。
    • 伪共享不命中（False Sharing Miss）：多个独立变量被划分在同一个 Cache 块中。一个核心修改了变量 A，导致持有同块中变量 B 的另一核心的整个 Cache 块被失效，后续读取 B 触发不命中（哪怕它根本不用 A ）。可通过 Cache 边界对齐或数据结构填充规避。 !!! warning "考试高频考点：伪共享（False Sharing）的物理本质与判定" 1. 物理本质：缓存一致性协议是以 Cache Line（缓存块，通常为 64 字节） 为单位进行失效和状态管理的，而不是以单个独立的变量为单位。 2. 判定依据：如果 P1 写了变量 A，导致 P2 在读写同一个 Cache Line 内的另一个不同变量 B 时发生不命中，这就属于伪共享不命中。如果是因为读写同一个变量导致的不命中，则是真共享不命中。 3. 软件层解法：通过编译器属性或手动填充（Padding）让不同线程访问的独立变量对齐存放在不同的 Cache Line 中。

    真伪共享不命中判定算例

    设 z1 和 z2 在同一个 Cache 块中，初始时在 P1 和 P2 均处于共享（Shared）状态：

    时间 处理器与操作 状态变化与动作描述 不命中类型归属
    1 P1 写 z1 块在 P2 中被失效，P1 转为 Modified 真共享不命中 (写共享块建立所有权)
    2 P2 读 z2 块被失效，P2 从 P1 获取数据，块在两核转为 Shared 伪共享不命中 (P2 读 z2 但其被 z1 的写入伪失效)
    3 P1 写 z1 块在 P2 中被失效，P1 转为 Modified 伪共享不命中 (P1 升级所有权但 P2 刚只访问了 z2)
    4 P2 写 z2 块在 P1 中被失效，P2 从 P1 获取并写入，P2 转为 Modified 伪共享不命中 (P2 因 z1 写入而不得不重新获取块所有权)
    5 P1 读 z1 块在 P2 被失效，P1 获取数据并读，转为 Shared 真共享不命中 (P1 读被 P2 写入修改过的新值 z2/z1)

    分布式目录协议机制

    目录协议通过在各节点主存中增加目录（Directory）避免了监听总线广播的物理瓶颈。

    1. 目录三状态

    • Shared（共享）：一个或多个核心缓存了该块，主存数据是最新的。
    • Uncached（未缓存）：没有核心缓存该块。
    • Modified（已修改）：只有一个核心拥有该块且已写入，主存数据已过期。

    2. 读写 Miss 消息控制流

    • 读不命中（Read Miss）：
    • 若目录为 Uncached/Shared：主节点从内存读取数据，发送给请求核心，目录设为 Shared 并记录共享节点。
    • 若目录为 Modified：主节点发送取回请求给占有核，占有核向主节点写回数据并发送给请求核，目录转为 Shared 并更新共享集。
    • 写不命中（Write Miss）：
    • 若目录为 Shared：主节点向所有共享核心发送失效信息（Invalidate），等待回复后将数据发送给写核心，目录转为 Modified 并记录独占核。
    • 若目录为 Modified：主节点将写不命中消息转发给占有核，占有核直接将块数据发给写核心并使自身失效，目录更新独占核。
