    第十二章 线程级并行与内存一致性

    本章分析了多处理器硬件同步原语的底层实现，探讨了利用缓存一致性优化自旋锁带宽的技术，系统比对了各种内存一致性模型（SC/TSO/PSO/WO/RC），并归纳了利用硬件投机来隐藏一致性延迟的方案。

    硬件同步与原子操作

    为了避免多个线程同时读写共享变量导致的数据竞争（Data Race），必须在硬件层面提供高可靠性的原子操作（Atomic Operations）以支撑互斥锁。

    1. 原子交换指令对：lr 与 sc

    RISC-V 采用 Load Reservation（预留加载）与 Store Conditional（条件存储）指令对来实现无锁原子交换： * lr rd, 0(rs1)：从内存地址 rs1 读取数据到寄存器 rd 中，并由硬件在该物理地址注册一个预留（Reservation）。 * sc rd, rs2, 0(rs1)：尝试将寄存器 rs2 的值写入内存地址 rs1。 * 成功：若此地址未被其他核心写入打断且未发生中断，则写入成功并在寄存器 rd 写入 0。 * 失败：若该地址已被修改或发生上下文切换，则预留被清空，写入失败并在 rd 写入非 0 值（如 1 ）。 * 硬件检测原理：处理器内部部署了专用的保留寄存器记录 lr 的物理地址。任何其他处理器发出的对该地址的写失效（Invalidation）信号、或者本地中断的发生，都会直接清除保留寄存器，从而导致后续的 sc 执行失败。

    2. 利用 lr/sc 实现原子交换的 RISC-V 汇编实例

    将寄存器 x4 的值与内存地址 0(x1) 处的锁状态原子交换，并将原状态返回给 x4： 

    try: addi x3, x4, 0      ; 将欲写入的值 x4 暂存到 x3
     lr.d x2, 0(x1)      ; 加载锁的值到 x2，并注册预留
     sc.d x3, x3, 0(x1)  ; 尝试将新值 x3 写入，成功则 x3 变 0 
     bnez x3, try        ; 若 x3 不为 0，说明被他人打断或失败，重新尝试
     addi x4, x2, 0      ; 将旧锁值 x2 拷贝给目标 x4，完成原子交换

    自旋锁与缓存一致性协同优化

    自旋锁（Spin Locks）是核心不断循环重试获取锁的一种忙等锁结构。

    1. 传统自旋锁的带宽瓶颈

    若直接使用原子交换宏 EXCH 不断循环读写锁状态：每次运行 EXCH（都执行了一次写尝试）会在总线上频繁广播写失效（Invalidate）信号，导致其他正在轮询的处理器 Cache 块不断失效并重新从主存拉取数据，产生极高的总线流量与主存竞争。

    2. 读自旋（Spin-on-Read）优化

    • 机制：核心在循环里先使用普通读指令 ld 持续查询本地 Cache 的锁状态（S 状态）。只要锁仍被他人持有（lock != 0），核心就只在本地 Cache 命中的只读循环里“自旋”，完全不产生总线流量。
    • 抢锁过程：一旦持有锁的核释放锁，会向总线广播 Invalidate 信号。轮询核的本地 Cache 被失效，随后的普通读 miss 并从主存重新获取锁的值为 0。此时各核心立即发起原子交换 EXCH 竞争锁。

    3. 基于 lr/sc 的轻量化自旋锁

    更高效的做法是利用 lr 替代普通读 ld，并将 EXCH 换为 sc： * 普通读即使拿到锁为 0，随后发起 EXCH 写动作依然无差别触发 Invalidate。 * 换用 lr/sc 对时，只有当某一个核抢锁时成功执行了 sc，才会真正向总线广播 Invalidate。抢锁失败核的 sc 默默写 1 失败并退回 lr 循环，完全不产生多余的总线写失效流量，最大限度保留了总线带宽。

    内存一致性模型与放松对比

    内存一致性模型（Memory Consistency Models）定义了多个不同物理地址的读写操作在多处理器环境下的全局可见顺序。 !!! warning "考试核心辨析：缓存一致性（Coherence） vs 内存一致性（Consistency）" 这两者在概念上极易混淆，但它们的关注角度截然不同，属于期末高频简答考点： 1. 缓存一致性（Coherence）：关注单个内存单元（单一地址）在多核 Cache 副本中的数据同步。它保证了“任何读操作都能读取到该地址最近写入的最新值”。 2. 内存一致性（Consistency）：关注多个不同内存单元（不同地址）在多核间的读写操作全局可见顺序。它定义了“当核心 P1 依次修改了变量 A 和 B，核心 P2 观察到 A 和 B 发生变化的先后次序”。 * 直观理解：缓存一致性负责确保同一个文件的本地拷贝不发生数据冲突；内存一致性则决定了在多核乱序执行下，多个不同读写操作的全局排列规则。

    1. 顺序一致性 (Sequential Consistency, SC)

    • 定义：任何程序执行的结果都与所有核按某种串行交错顺序访存且各核内部执行均严格保持程序顺序（Program Order）的结果相同。
    • 硬件代价：为了维护 SC，任何写操作必须阻塞，直到总线上所有的写失效信号被其他核心完全接收并确认回复（Acknowledge）之后，写操作才算彻底完成。这阻碍了动态调度的乱序执行。

    2. 放松一致性模型对比

    由于 SC 代价高昂，现代处理器根据读写次序（\(R \to R\)、\(R \to W\)、\(W \to R\)、\(W \to W\)）的放松程度演进出多种弱一致性模型：

    内存一致性模型 放松的读写物理顺序 对同步指令（Sync）的约束要求 典型代表硬件
    顺序一致性（SC） 无（最严格） 所有读写均需要维持绝对顺序 早期学术模型
    全店序（TSO） 放松 \(W \to R\) (写后面的读可先做) 必须保证同步读写与屏障的顺序 x86 架构
    偏店序（PSO） 放松 \(W \to R\) 与 \(W \to W\) 必须使用屏障约束同地址写的先后 SPARC 架构
    弱一致性（WO）/ 释放一致性（RC） 放松所有四种普通读写顺序 普通读写可以任意乱序，只约束同步指令之间的顺序 RISC-V, ARM, PowerPC

    释放一致性协议 (Release Consistency)

    释放一致性（RC）是弱一致性的一种典型形式，将同步操作细分为获取（Acquire）与释放（Release）：

    • 获取（Acquire, SA）：表示进入临界区获取共享数据访问权。
    • 重排界限：在 SA 之后的任何普通读写操作，禁止被重排到 SA 之前执行。
    • 释放（Release, SR）：表示退出临界区释放访问权。
    • 重排界限：在 SR 之前的任何普通读写操作，必须全部完成，禁止被重排到 SR 之后。
    • 物理意义：SA 之前的普通读写和 SR 之后的普通读写可以自由跨越，赋予了流水线极高乱序执行空间。

    硬件投机隐藏一致性延迟

    动态调度处理器可以利用硬件投机（Speculation）在保障严格一致性模型（如 SC）正确输出的前提下，通过乱序执行读写来掩盖长延迟：

    • 延迟提交（Delayed Commit）：允许读指令在 ROB 中投机式提前乱序执行。
    • 投机恢复（Speculation Recovery）：在投机读取数据（未提交）期间，如果核心从总线上收到了其他处理器对该数据地址的写失效（Invalidate）广播，说明当前投机读取的数据已被破坏。
    • 此时核心利用 ROB 中的控制信息触发恢复机制，回滚并放弃自该访存指令后的所有投机计算结果，并从该被失效的访存地址处重新执行，完美保证了顺序一致性行为。
