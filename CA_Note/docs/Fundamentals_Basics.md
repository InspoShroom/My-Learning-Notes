# 第二章 计算机设计基础

本章详细介绍了定量计算机设计的基础知识，包括处理器性能演变规律、定量分析的核心定律、计算机分类、并行性的 Flynn 分类法，以及 ISA 设计的七个关键维度。

---

## 处理器性能的发展

过去四十年处理器性能的飞速发展，主要得益于微结构设计的突破与半导体工艺的红利。

1. **摩尔定律（Moore's Law）**：集成电路上可容纳的晶体管数目大约每两年增加一倍。该定律在 2015 年左右因达到原子级物理极限而逐渐终结。
2. **登纳德缩放定律（Dennard Scaling）**：随着晶体管尺寸缩小，其功耗密度保持恒定。因此可以在保持芯片总功耗和散热不变的前提下，大幅提升主频和晶体管数量。
    * **终结原因**：约在 2004 年终结。由于工作电压（Voltage）和漏电流（Current）无法继续等比例降低，芯片功耗密度急剧上升，主频被迫封顶（“功耗墙”（Power Wall））。
3. **多核转型**：物理极限迫使产业界转向**多核处理器（Multi-Core Processor）**，通过横向增加高效核心，将性能提升的途径从发掘指令级并行（ILP）转向发掘数据级并行（DLP）与线程级并行（TLP）。

## 定量设计准则：阿姆达尔定律与 Lhadma 定律

### 1. 阿姆达尔定律（Amdahl's Law）

阿姆达尔定律用于计算对系统某一部分进行优化后，系统整体所获得的性能提升/加速比。

* **加速比公式**：
  ${Speedup} = \frac{1}{(1 - F_{	ext{enhanced}}) + \frac{F_{	ext{enhanced}}}{S_{	ext{enhanced}}}}$
  其中：
    * $F_{	ext{enhanced}}$：被优化的部分在原系统运行时间中所占的比例（非负且 $\le 1$）。
    * $S_{	ext{enhanced}}$：该优化部分自身提升的加速比。
* **加速比极限**：
  当优化部分的性能提升趋近于无穷大（$S_{	ext{enhanced}} 	o \infty$）时，整体加速比的极限为：
  $ext{Limit} = \frac{1}{1 - F_{	ext{enhanced}}}$
    * *启示*：若一个任务中有 10% 的部分无法被并行（必须串行执行），那么即使投入无限个核心，系统的最大加速比也只有 10 倍。
* **设计准则**：**使常见情况变快（Make common case fast）**。应优先优化占比高的部分，以获得最大的整体收益。
    * *典型例题*：假设某一处理器的浮点求平方根指令（FPSQR）执行时间占总时间的 $20\%$。通过硬件重构，可将该指令的执行速度提升至 $10$ 倍。
        1. 若仅优化此指令，系统整体的加速比是多少？
        2. 另一备选方案是优化所有浮点运算指令（占总执行时间的 $50\%$），使其整体速度提升至 $2$ 倍。两方案哪个更优？
    * *解答*：
        1. 仅优化平方根指令：$F_{	ext{enhanced}} = 0.20$，$S_{	ext{enhanced}} = 10$。
           $ext{Speedup}_{1} = \frac{1}{(1 - 0.20) + \frac{0.20}{10}} = \frac{1}{0.80 + 0.02} \approx 1.22$
           整体性能提升约 $22\%$。
        2. 优化全部浮点指令：$F_{	ext{enhanced}} = 0.50$，$S_{	ext{enhanced}} = 2$。
           $ext{Speedup}_{2} = \frac{1}{(1 - 0.50) + \frac{0.50}{2}} = \frac{1}{0.50 + 0.25} \approx 1.33$
           整体性能提升约 $33\%$。
        * **结论**：优化占比更高（$50\%$）但提升温和（$2$ 倍）的浮点操作，其带来的收益高于优化占比低（$20\%$）但提升巨大（$10$ 倍）的平方根操作。这充分印证了“使常见情况变快”的设计哲学。

### 2. Lhadma 定律

作为阿姆达尔定律的补充，Lhadma 定律强调：**不能让非常规（未优化）的情况变得太慢（Yet uncommon case not so slow）**。

* *反例计算*：如果对占比 90% 的部分加速了 2 倍（$S_{	ext{enhanced}} = 2$），但作为代价，使得剩下的 10% 部分慢了 10 倍（相当于加速比为 $0.1$）。整体加速比计算为：
  $ext{Speedup} = \frac{1}{\frac{0.1}{0.1} + \frac{0.9}{2}} = \frac{1}{1 + 0.45} \approx 0.69$
  结果显示，由于未优化的部分变慢，整体性能反而退化了 31%。

## 计算机的五大分类

现代计算机系统根据应用场景和约束条件的不同，主要分为以下五大类：

| 分类 | 典型设备 | 核心设计指标 | 主要约束/瓶颈 | 特征 |
| :--- | :--- | :--- | :--- | :--- |
| 个人移动设备（PMD） | 手机、平板 | 响应速度、能效 | 电池电量、散热（无风扇） | 强响应性，属于软实时（Soft Real-Time）系统，注重代码体积优化 |
| 桌面计算（Desktop） | 个人电脑、工作站 | 性价比（Price-Performance） | 整机成本、性能表现 | 最大的单一市场，最注重单机计算能力与图形表现 |
| 服务器（Servers） | 数据中心服务器 | 可用性（Availability）、吞吐量 | 宕机经济损失、扩展成本 | 强调防失效的可用性，以及高可扩展性（Scalability）和高吞吐量 |
| 仓储级计算机（WSC） | 云计算数据中心 | 性价比、能源效率 | 电力损耗、建设成本 | 强调交互式服务、海量存储与极高的网络带宽（不同于超算专注于浮点运算） |
| 嵌入式计算机（Embedded） | 智能家居、物联网设备 | 最低价格满足性能 | 极度敏感的价格与功耗限制 | 隐藏在设备中，其核心分界线在于是否具备运行第三方软件的能力 |

## 并行性的层级与 Flynn 分类法

### 1. 并行性的层级
* **应用级并行（Application Parallelism）**：
    * **数据级并行（Data-Level Parallelism, DLP）**：同时对许多独立的数据项执行相同的操作。
    * **任务级并行（Task-Level Parallelism, TLP）**：构建多个独立的任务流以并行方式运行。
* **硬件发掘并行机制**：
    * **指令级并行（Instruction-Level Parallelism, ILP）**：通过流水线和投机执行等在硬件级发掘细粒度的并行性。
    * **SIMD/向量架构/GPU**：硬件执行单条指令，对数据集合进行并行操作，发掘数据级并行（DLP）。
    * **线程级并行（Thread-Level Parallelism, TLP）**：利用多核硬件模型发掘 DLP 或 TLP。
    * **请求级并行（Request-Level Parallelism, RLP）**：利用解耦的硬件节点并行处理大量独立的客户端请求。

### 2. Flynn 分类法
基于指令流（Instruction stream）和数据流（Data stream）的数量，将计算机体系结构划分为四类：

* **SISD（Single Instruction, Single Data）**：单指令流单数据流。传统单核 CPU，主要发掘指令级并行（ILP）。
* **SIMD（Single Instruction, Multiple Data）**：单指令流多数据流。相同的指令在多个处理单元上执行，操作不同的数据流（发掘 DLP，如向量架构、GPU、多媒体指令集）。
* **MISD（Multiple Instruction, Single Data）**：多指令流单数据流。目前没有主流的商业实现。
* **MIMD（Multiple Instruction, Multiple Data）**：多指令流多数据流。每个处理器自主获取指令并操作各自的数据。具有极高灵活性，是多核处理器与分布式系统的核心（发掘 TLP 与 RLP）。

## ISA设计的七个维度

指令集体系结构（ISA）是软件与硬件的接口交界面。定义一个 ISA 必须确定以下七个维度的属性：

### 1. ISA 的分类 (Class of ISA)
* **寄存器-内存型（Register-Memory ISA）**：许多指令可以直接将内存地址作为操作数，如 80x86。
* **加载-存储型（Load-Store / Register-Register ISA）**：只有专属的 Load 和 Store 指令可以访问内存，其余所有计算指令的操作数必须是寄存器，如 RISC-V 和 ARMv8。
  * *趋势*：1985 年之后设计的所有现代 ISA 全都采用了 Load-Store 架构。

### 2. 内存寻址与对齐 (Memory Addressing & Alignment)
* **字节寻址（Byte Addressing）**：内存中最小的寻址单元是一个字节（Byte, 8 bits）。
* **内存对齐（Aligned Address）**：当数据对象在内存中的物理起始地址 $A$ 满足 $A \mod s = 0$（$s$ 为对象字节宽度）时，称为地址对齐。
* **性能影响**：未对齐 of 访存（Misaligned access）在硬件上需要发起两次内存访问并进行数据拼接，会导致显着的性能损耗。因此，许多 ISA 强制或建议数据在内存中按其宽度边界对齐放置。

### 3. 寻址方式 (Addressing Modes)
用于指明指令中的操作数存放在哪里（寄存器、常数/立即数、还是内存地址）。

* **RISC-V** 仅支持 3 种基本寻址：寄存器（Register）、立即数（Immediate，如 `addi x1, x1, 16`）、偏移量寻址（Displacement，如 `lw x1, 100(x2)`）。

??? example "RISC-V 寻址方式详解与对比"
    ### RISC-V 的 3 种基本寻址方式
    1. **寄存器寻址（Register Addressing）**
        * *特点*：操作数直接存放在寄存器中，指令中指定寄存器编号。
        * *汇编*：`add x1, x2, x3`
        * *执行过程*：将寄存器 `x2` 的值与 `x3` 的值相加，结果存回寄存器 `x1`。完全在 CPU 内部的寄存器堆和 ALU 内完成，无内存访问。
    2. **立即数寻址（Immediate Addressing）**
        * *特点*：操作数是一个常量，作为指令的一部分直接编码。
        * *汇编*：`addi x1, x2, 100`
        * *执行过程*：将寄存器 `x2` 的值与立即数常量 `100` 相加，结果存回 `x1`。常量 `100` 直接从指令码中解析出，无需访存。
    3. **偏移量寻址 / 寄存器相对寻址（Displacement Addressing）**
        * *特点*：操作数在内存中，物理地址由基址寄存器内容与指令中指定的立即数偏移量相加得到。
        * *汇编*：`lw x1, 8(x2)`
        * *执行过程*：计算访存物理地址 $Addr = x2 + 8$，然后从内存该地址处读取一个字并载入到寄存器 `x1`。这是 RISC-V 唯一的内存访存寻址方式。

    ---

    ### 复杂 ISA 寻址方式对比（RISC-V 不支持）
    为简化译码和流水线设计，RISC-V 剔除了复杂指令系统中的高级寻址模式，需要通过多条简单指令组合实现相同功能：
    * **比例变址寻址（Scaled Index，如 x86）**
        * *x86 汇编*：`mov eax, [ebx + ecx * 4]` （常用于数组快速访问）
        * *RISC-V 等价实现*：需先用逻辑左移计算偏移量，再与基址相加，最后访存：
          ```assembly
          slli x4, x3, 2     # x3 (ecx) 左移 2 位 (乘以 4)，结果存入临时寄存器 x4
          add  x4, x2, x4     # 将基址 x2 (ebx) 与偏移量相加
          lw   x1, 0(x4)      # 从最终地址载入数据
          ```
    * **自增寻址（Autoincrement，如 ARM）**
        * *ARM 汇编*：`ldr r1, [r2], #4` （访存后自动递增指针，常用于循环）
        * *RISC-V 等价实现*：必须将访存与基址指针更新分两步执行：
          ```assembly
          lw   x1, 0(x2)      # 载入基地址数据
          addi x2, x2, 4      # 将指针 x2 递增 4 字节
          ```

* **80x86 / ARMv8** 则支持更多复杂的变体：如比例变址（Scaled index，用于快速访问数组元素）、自增/自减（Autoincrement/Autodecrement，在访存的同时修改基址寄存器的值）等。

### 4. 操作数类型与大小 (Types and Sizes of Operands)
指明硬件支持的数据宽度：如字符（8位）、半字（16位）、字（32位）、双字（64位），以及单精度/双精度浮点数等。

### 5. 操作指令分类 (Operations)
主要分为四大通用类别：数据传输（Data transfer）、算术与逻辑运算（Arithmetic/Logical）、控制流指令（Control）、以及浮点运算（Floating point）。

### 6. 控制流指令 (Control Flow Instructions)
包括条件分支（Conditional branches）、无条件跳转（Unconditional jumps）、过程调用（Procedure calls）与返回（Returns）。

* **返回地址存储**：过程调用指令通常采用**跳转并链接（Jump and Link）**机制，在跳转的同时将紧随其后的下一条顺序指令的物理地址保存到链接寄存器（Link Register, 如 RISC-V 的 `ra` 寄存器）中，以便子程序执行完毕后能直接返回。

### 7. 指令集编码 (Encoding an ISA)
确定如何将指令的语义和操作数转换为二进制格式。

* **固定长度编码（Fixed Length）**：所有指令长度完全相同（如 RISC-V / ARMv8 均为 32 位）。便于硬件高速译码与设计流水线，但略微牺牲了程序代码体积。
* **可变长度编码（Variable Length）**：指令长度不固定（如 80x86 介于 1 到 18 字节之间）。虽然能显著压缩代码体积，但导致译码硬件极其复杂，难以发掘 ILP。
